# Roster Access Controls Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make roster membership status the default source of truth for existing login access, preserve explicit admin overrides, and require confirmation before imports that would remove more than 10 members.

**Architecture:** Add explicit override state to `User`, extend `RosterImport` with a pending-confirmation status and an attached pending CSV, centralize roster-controlled login policy on `User`, and teach `RosterImports::Importer` to preflight removals before applying. Admin account actions set or clear overrides; imports respect overrides and report access effects in the import summary.

**Tech Stack:** Rails 8.1, Active Record, Active Storage, Minitest, ERB, existing Hotwire/Tailwind app shell.

---

## File Structure

- `db/migrate/*_add_roster_access_controls.rb` — adds user override columns and permits pending import status support through existing `roster_imports.status` string.
- `app/models/user.rb` — owns login override state, normalized roster-status policy, and last-admin-safe roster-controlled enable/disable.
- `app/models/roster_import.rb` — adds `pending_confirmation` status, `has_one_attached :pending_csv`, access-effect readers, and exception helpers.
- `app/services/roster_imports/importer.rb` — computes removals before mutation, creates pending imports for large removals, applies status-driven access, and writes structured summary counts.
- `app/controllers/admin/roster_imports_controller.rb` — adds confirm action and required checkbox handling.
- `app/controllers/admin/user_accounts_controller.rb` — admin enable/disable sets override; revert clears override and reapplies roster policy.
- `app/views/people/_login_account.html.erb` — displays roster-controlled/admin-override state and revert action.
- `app/views/admin/roster_imports/show.html.erb` — renders pending-confirmation UI, access-effect summary, and sign-in exceptions.
- Existing tests under `test/models`, `test/services/roster_imports`, and `test/controllers/admin` cover the behavior.

```ruby
class RosterImport < ApplicationRecord
  STATUSES = %w[completed failed pending_confirmation].freeze
  STALE_AFTER = 30.days

  has_one_attached :pending_csv

  validates :uploaded_filename, :status, :imported_at, presence: true
  validates :status, inclusion: { in: STATUSES }
  validates :created_count, :updated_count, :unchanged_count, :removed_count, :problem_count,
    numericality: { greater_than_or_equal_to: 0 }

  scope :successful, -> { where(status: "completed") }
  scope :history, -> { order(imported_at: :desc, id: :desc) }

  def self.latest_successful
    successful.order(imported_at: :desc, id: :desc).first
  end

  def self.roster_stale?
    latest = latest_successful
    latest.blank? || latest.imported_at < STALE_AFTER.ago
  end

  def problems
    Array(summary&.fetch("problems", nil))
  end

  def removed_members
    Array(summary&.fetch("removed_members", nil))
  end

  def access_effects
    summary&.fetch("access_effects", {}) || {}
  end

  def sign_in_exceptions
    User.where(login_access_override: true).includes(:person).order("people.last_name", "people.first_name")
  end
end
```

- [ ] **Step 3: Implement the policy methods**

Add to `app/models/user.rb` after `only_enabled_administrator?`:

```ruby
  ROSTER_LOGIN_ENABLED_STATUSES = %w[active grace].freeze
  ROSTER_LOGIN_DISABLED_STATUSES = %w[expired deceased].freeze

  def apply_roster_access!
    return :skipped_admin_override if login_access_override?

    apply_roster_access_without_override_check!
  end

  def return_to_roster_control!
    return :skipped_last_admin if roster_access_should_disable? && only_enabled_administrator?

    update!(login_access_override: false, login_access_override_at: nil)
    apply_roster_access_without_override_check!
  end

  def set_login_access_override!(disabled:)
    update!(disabled_at: (disabled ? Time.current : nil), login_access_override: true, login_access_override_at: Time.current)
  end

  def roster_access_status
    return "removed" if person.roster_removed_at.present?

    person.roster_member_status.to_s.strip.downcase
  end

  def roster_access_should_enable?
    ROSTER_LOGIN_ENABLED_STATUSES.include?(roster_access_status)
  end

  def roster_access_should_disable?
    roster_access_status == "removed" || ROSTER_LOGIN_DISABLED_STATUSES.include?(roster_access_status)
  end

  def roster_access_unsupported_status?
    !roster_access_should_enable? && !roster_access_should_disable?
  end

  private

  def apply_roster_access_without_override_check!
    return :unsupported_status if roster_access_unsupported_status?

    if roster_access_should_enable?
      update!(disabled_at: nil) if disabled_at.present?
      :enabled_by_roster_status
    elsif only_enabled_administrator?
      :skipped_last_admin
    else
      update!(disabled_at: Time.current) if disabled_at.blank?
      :disabled_by_roster_status
    end
  end
```

Keep the existing `private` section and `assign_webauthn_id` below these methods by placing the new `private` before `apply_roster_access_without_override_check!` and leaving `assign_webauthn_id` private.

```ruby
module Admin
  class UserAccountsController < BaseController
    def create
      @person = Person.find(params[:person_id])
      email_address = params.dig(:user, :email_address).presence || @person.roster_email_address

      if email_address.blank?
        redirect_to person_path(@person), alert: "Enter a login email address before creating the account."
        return
      end

      if (user = @person.user)
        user.update!(email_address: email_address)
        user.set_login_access_override!(disabled: false)
      else
        User.create!(
          person: @person,
          email_address: email_address,
          email_verified_at: Time.current,
          login_access_override: true,
          login_access_override_at: Time.current
        )
      end

      redirect_to person_path(@person), notice: "Login account is enabled as an admin exception."
    rescue ActiveRecord::RecordInvalid => e
      redirect_to person_path(@person), alert: e.record.errors.full_messages.to_sentence
    end

    def destroy
      @person = Person.find(params[:person_id])
      user = @person.user

      if user&.disabled_at.blank? && user.can?("manage_settings") && !User.another_enabled_manage_settings_user_exists?(user)
        redirect_to person_path(@person), alert: "At least one enabled administrator account is required."
        return
      end

      user&.set_login_access_override!(disabled: true)

      redirect_to person_path(@person), notice: "Login account is disabled as an admin exception."
    end

    def roster_control
      @person = Person.find(params[:person_id])
      user = @person.user

      if user.blank?
        redirect_to person_path(@person), alert: "There is no login account to return to roster control."
        return
      end

      result = user.return_to_roster_control!
      if result == :skipped_last_admin
        redirect_to person_path(@person), alert: "At least one enabled administrator account is required."
      else
        redirect_to person_path(@person), notice: "Login account now follows roster status."
      end
    end
  end
end
```

      imported_member_numbers = rows.map(&:member_number)

    def pending_import(rows, row_problems)

      roster_import = RosterImport.find(params[:id])
      result = nil
      alert = nil

      roster_import.with_lock do
        roster_import.reload

        if roster_import_superseded?(roster_import)
          alert = "That roster import can no longer be confirmed."
        elsif roster_import.status != "pending_confirmation" || roster_import.pending_csv.blank? || !params[:confirm_large_removal].present?
          alert = roster_import.status == "pending_confirmation" ? "Confirm the large removal before applying this import." : "That roster import can no longer be confirmed."
        else
          importer = RosterImports::Importer.new(csv_text: roster_import.pending_csv.download, filename: roster_import.uploaded_filename, roster_import: roster_import, confirm_large_removal: true)
          result = importer.import
          alert = result.success? ? nil : "Roster import could not be completed."
        end
      end

      if result&.success?
        redirect_to admin_roster_import_path(result.roster_import), notice: "Roster import completed."
      else
        redirect_to admin_roster_import_path(roster_import), alert: alert || "That roster import can no longer be confirmed."
      end
    end

    end
